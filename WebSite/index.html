Introduction

i) Introduction and database description.
i) The main goal of the project is the management of the university database. The aim of this project is to develop software for the effective management of a university. This DBMS will be used by University's faculties. Staffs can use this DBMS for maneging, accessing and updating infomation about students, proforessors, courses, libraries and so on.

ii) What functions should the system perform? For example, inventory control, billing, ordering, etc.
ii) The System can find, update, modify and delete information from the university database. For example: We can easily find infomation about what course student is in or what their grad is. Also which professor teaches which course. In additions we can access data on the library, assistants, departments, sections, courses etc.

iii) Who are the end users? Remember that the DBA is NOT an end user.
iii) "End users are basically those people whose jobs require access to the database for querying, updating, and generating reports. The database primarily exists for their use". As i mentioned before, end users can be staffs from faculties and also 'Student help center'.

iv) How will data obsolescence be handled?
iv) To not lose any data from database, I'm using Triggers, Transactions and solid structure of code. Also my tables are in BCNF, which means no anomalies in my database.
